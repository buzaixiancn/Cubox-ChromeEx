## 📋 使用说明

### 文件夹命名规范

1. **一级分类使用中文名称**，清晰明确
2. **二级分类使用中文或英文**，保持一致性
3. **特殊分类**：
   - `收集箱` - 默认收件箱，新收藏内容会先进入这里
   - `已归档书签` - 归档的旧书签
   - `测试文件夹` - 测试用途

### 分类建议

#### 编程相关
- **编程** - 通用编程技术、代码仓库
- **前端** - 前端开发相关（Vue、React、JavaScript等）
- **开源** - 开源项目、开源工具

#### 跨境电商
建议按照业务流程组织：
- **工具使用** - 常用工具和平台使用指南
- **Tiktok** - TikTok运营、工具、API
- **独立站** - Shopify、WooCommerce等独立站建站
- **SEO** - SEO优化、关键词研究
- **选品** - 产品选择、市场分析
- **收款** - 支付收款解决方案
- **邮件营销** - 邮件营销工具和策略
- **广告** - 广告投放、广告平台

#### 设计相关
- **设计参考** - 设计灵感和参考案例
- **Figma** - Figma插件和资源
- **UI/UX** - UI/UX设计资源和工具

### 标签使用建议

除了文件夹分类，还可以配合标签使用：
- 技术标签：`GitHub`、`Vue`、`Python`、`JavaScript`
- 平台标签：`TikTok`、`Shopify`、`WordPress`
- 类型标签：`代码仓库`、`工具`、`教程`、`API`

## 📝 注意事项

1. **文件夹不存在会自动创建**：使用 Cubox API 收藏时，如果指定的文件夹不存在，系统会自动创建。

2. **留空则进入收集箱**：如果不指定 `folder` 参数，内容会默认收藏到"收集箱"。

3. **嵌套文件夹**：Cubox 支持嵌套收藏夹，可以使用 `/` 分隔，如：`跨境电商/Tiktok`。

4. **定期整理**：建议定期整理和归档收藏内容，保持分类清晰。
